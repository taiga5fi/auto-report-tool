## 成功する図解の構造

```
1. ヘッダー
   └─ 概念名 + 本質を一行で + 「直感→仕組み→理論→誤解」の案内

2. まず直感でつかむ（アナロジー2〜3個）
   └─ 日常生活の例
   └─ IT/システムの例（この読者には効果的）
   └─ 概念の構成要素との対応表

3. 仕組みを理解する（因果フロー）
   └─ 原因→この概念→結果 の関係図
   └─ ステップ解説（多くて3〜4ステップ）

4. 理論・数式を読む（概念に数式がある場合のみ）
   └─ 記号の意味を先に説明
   └─ 数式全体を日本語で言い換え

5. よくある誤解（必須・最重要）
   └─ 誤解が生まれる理由を明示
   └─ 誤解（赤）→ 正しい理解（緑）の訂正セット

6. 人への説明まとめ
   └─ 30秒バージョン（そのまま使える文）
   └─ 理解度チェック（自問自答）
```

## アナロジーの選び方

### 良いアナロジー
- **IT/システムの概念**: この読者はプログラム・ネットワーク・サーバーを知っているので有効
- **日常の物理現象**: 水の流れ、重力、天秤など直感的なもの
- **座標・基準点**: 「ゼロ点を決める」概念に使える

### アナロジーの対応表を必ず書く

```
例え話「〇〇（例）」との対応:
電力系統の概念A ↔ 例え話の要素X
電力系統の概念B ↔ 例え話の要素Y
```

対応表がないと「例え話はわかったけど概念に戻れない」が起きる。

### アナロジーの失敗パターン

```
❌ 悪い: 「スイングノードは指揮者のようなものです」（類似性が曖昧）
✅ 良い: 「スイングノードはGPSの基準点（0,0）のようなものです。
         位置情報の計算では絶対的な原点が必要なのと同様、
         電力系統の潮流計算では位相角の基準点が必要です。
         基準点は測定結果を受け取る側ではなく、計算の出発点です。」
```

## よくある誤解の書き方（最重要）

### 構成の鉄則

1. **誤解が生まれる理由**を書く（「〇〇という言葉が〜に聞こえるから」）
2. **誤解の内容**を具体的に書く
3. **正しい理解**を書く
4. **なぜ誤解が間違いなのか**の根拠を書く

### 例: スイングノード（ヒアリングシートの事例）

```
誤解が生まれる理由:
「スイング（揺れる）」という言葉と「誤差を吸収する」という動詞の組み合わせから、
「スイングノードが誤差を受け取って処理する」と連想されやすい。

誤解: 「潮流誤差はスイングノードで吸収されるから需給バランスは一致する」

正しい理解:
スイングノードは「位相角 = 0」に固定された基準点（ゼロ基準）であり、
計算結果の誤差を"受け取って消す"ノードではない。
スイングノードの有効電力出力は計算の過程で「最後に決まる変数」であり、
系統全体の需給を合わせるために自動調整される。

根拠:
潮流計算では全ノードの電圧・位相角・電力を同時に求めるが、
位相角は相対値なので基準（0点）が必要。スイングノードはその基準を提供する。
誤差が出る場合、それはモデルの精度や収束問題であり、
スイングノードが"消す"仕組みはない。
```

## 数式解読の書き方

### 鉄則: 記号の意味を先に説明する

```
❌ 悪い: 「P = V × I × cos(θ) です」（記号の意味が不明）

✅ 良い:
// 各記号の意味
P = 有効電力（実際に仕事をする電力、単位はW）
V = 電圧（電気を押し出す圧力）
I = 電流（電気の流れ量）
θ = 電圧と電流の位相差（ずれの角度）
cos(θ) = 力率（どれだけ有効に使えているかの割合）

// 数式全体を日本語で
P = V × I × cos(θ)
「実際に使える電力 = 電圧 × 電流 × 有効利用率」
```

### 数式がない場合はスキップしてよい

電力市場制度など、数式なしで説明できる概念には数式セクション不要。

## 「説明できる状態」の判断基準

図解を読んだ後に以下ができれば成功:

| チェック | 問い |
|---------|------|
| 本質を言える | 「〇〇とは一言で言うと？」に答えられる |
| 誤解を訂正できる | 「〇〇は△△するのでは？」という問いに「違います、理由は〜」と言える |
| アナロジーで説明できる | 技術用語を使わずに例え話で説明できる |
| 関係概念を説明できる | 「〇〇と△△の違いは？」に答えられる |

## 品質チェックリスト

### 必須

- [ ] ヘッダーに「本質を一行で」書いてある
- [ ] アナロジーが2つ以上あり、概念との対応表がある
- [ ] 仕組みの因果フロー（原因→概念→結果）がある
- [ ] **よくある誤解**セクションがある（誤解の理由・誤解内容・正しい理解・根拠の4点）
- [ ] 「30秒バージョン」の説明まとめがある
- [ ] 理解度チェック（自問自答）がある

### 数式がある場合

- [ ] 記号の意味を先に個別に説明している
- [ ] 数式全体を日本語で言い換えている
- [ ] 数式の「意味」（何を表しているか）を説明している

### デザイン

- [ ] Lucide iconを使用（絵文字禁止）
- [ ] アナロジー（暖色）と仕組み（青）で色を使い分けている
- [ ] 誤解（赤）→ 正しい理解（緑）の訂正セットになっている
- [ ] スマホでも読みやすい（レスポンシブ対応）

### 読みやすさ

- [ ] 電力理論・数学の知識がゼロでも理解できるか
- [ ] 直感→仕組み→理論の順で読める構造になっているか
- [ ] 読み終えて「説明できる気がする」状態になるか
