# Diagram Power Concept

電力系統・電力市場の概念を**自分（電力理論・数学・制度の知識が低いSE/PL）**が理解し、人に説明できる状態になるHTMLに変換する。

## 想定読者

- **自分自身**（業務歴8年だが電力理論・数学・制度の知識は低い）
- ゴール: 図解を見て「理論的にかつビジュアル的に理解し、**人に説明できる状態**」になること

---

## ワークフロー

### Phase 0: 概念の把握

入力を整理する:

- **概念・疑問**: 何を理解したいか（用語・現象・仕組み）
- **背景**: どういう文脈で出てきたか（会議・仕様書・教科書など）
- **つまずき点**: 何が「わからない」のか（「〇〇と△△の違い」「なぜ〜なのか」など）
- **誤解の疑い**: 「〇〇だと思っていたが違うかもしれない」という仮説があれば

### Phase 1: 徹底調査（サブエージェント）

**exploreサブエージェント**を起動して、対象概念を徹底調査する。

```
Task({
  subagent_type: "explore",
  description: "電力系統概念の徹底調査",
  prompt: `
    以下の電力系統・電力市場の概念を徹底的に調査してください：
    【対象】{概念・疑問}
    【文脈】{背景}
    
    調査項目：
    1. この概念の本質的な定義（1〜2文）
    2. よくある誤解・混同パターン（特に重要）
    3. 直感的に理解できるアナロジー（日常生活や IT での例）
    4. 数式・理論の意味（数式は記号の意味から説明）
    5. 関連概念との関係（何が原因で何が結果か）
    6. 実際の電力系統でどう使われるか（実用例）
    
    結果は箇条書きで整理して返してください。
  `
})
```

### Phase 2: アナロジーを3つ考える

調査結果から、以下のパターンでアナロジーを選ぶ:

| 種類 | 例 |
|------|-----|
| 日常生活の例 | 水道・交通・重力など |
| IT/システムの例 | サーバー・ネットワーク・プログラムなど |
| 数学/物理の例 | 釣り合い・座標・基準点など |

### Phase 3: 説明の流れを設計する

「直感 → 仕組み → 理論 → よくある誤解」の順で構造を設計する:

1. **直感（アナロジー）**: 知っているものに例える
2. **仕組み**: 何が何をしているか（因果関係）
3. **理論・数式**（ある場合）: 記号の意味から順に説明
4. **よくある誤解**: この概念で間違えやすいポイントと正しい解釈
5. **人への説明まとめ**: 30秒で説明するなら何と言うか

### Phase 4: HTML生成

直感からの理解を積み上げる構造でHTMLを生成する。

**詳細** → [references/html-structure.md](references/html-structure.md)

### Phase 5: 批判的レビュー（サブエージェント）

**電気理論・数学の知識がほぼゼロだがIT経験はある人の視点**で、忖度なくレビューする。

```
Task({
  subagent_type: "generalPurpose",
  description: "概念図解の批判的レビュー",
  readonly: true,
  prompt: `
    あなたは「IT/システム開発の経験は10年あるが、電力・電気理論・数学の知識はほぼゼロ」の人物です。
    
    以下のHTML図解を読んで、**忖度なく**批判してください：
    
    【HTMLファイルパス】{path}
    
    ## レビュー観点
    
    1. **わからない言葉**: 説明なしに出てくる電力・数学用語はどれか
    2. **アナロジーの有効性**: 例え話がピンとくるか、逆に混乱するか
    3. **理解の積み上げ**: 前の説明を理解してないと次が読めない箇所はないか
    4. **よくある誤解の説明**: 「なぜその誤解が生まれるか」まで書かれているか
    5. **説明できるか**: 読み終えて「30秒で人に説明できる気がするか」
    
    ## 出力形式
    
    ### 致命的な問題（これがあると理解できない）
    - ...
    
    ### 改善すべき点（直せばもっと良くなる）
    - ...
    
    ### 良かった点（このまま残すべき）
    - ...
    
    ### 総評
    5段階評価: ⭐⭐⭐☆☆（3/5）のように
    一言: 「○○だから△△は理解できるが〜は難しい」のように
  `
})
```

### Phase 6: ブラッシュアップ

Phase 5のレビュー結果を踏まえてHTMLを修正する。

**優先度**:
1. 致命的な問題 → 必ず修正
2. 改善すべき点 → 可能な限り対応
3. 良かった点 → 維持

### Phase 7: デプロイ

```bash
# Windows環境ではGit BashまたはWSLで実行する
DEPLOY_DIR=$(mktemp -d)
# index.html を $DEPLOY_DIR に書き込む
cd "$DEPLOY_DIR" && surge . --domain {概念名をkebab-caseにしたもの}-concept.surge.sh
```
